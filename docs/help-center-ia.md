# Help Center Information Architecture

## Objective

Provide in-app documentation that is concise, operationally useful, and aligned with enterprise-quality desktop software standards.

## Section Model

1. **Overview**
- Product purpose
- Version/build identity

2. **Getting Started**
- Workspace setup
- First-note workflow

3. **Core Workflows**
- Authoring and validation
- View controls and navigation
- History and restore

4. **Advanced Capabilities**
- Media operations
- AI operations
- P2P operations

5. **Support and Troubleshooting**
- Common recovery actions
- Escalation paths for build/runtime failures

## UX Principles

- Use descriptive headings and imperative actions.
- Keep paragraphs short and task-oriented.
- Prefer explicit menu paths and shortcut references.
- Ensure version/build identity is always visible for support diagnostics.
